@title = 'Αποστολή Mail'

Για αποστολή αλληλογραφίας, πρέπει να έχετε ρυθμίσει σωστά τις ρυθμίσεις SMTP.

h2. Τι είναι το SMTP;

Το SMTP σημαίνει Simple Mail Transfer Protocol. Είναι ο τρόπος με τον οποίο οι διακομιστές (servers) αλληλογραφίας συνομιλούν μεταξύ τους για την παράδοση αλληλογραφίας. Εάν θέλετε να στείλετε οποιοδήποτε mail στο διαδίκτυο, χρειάζεστε έναν διακομιστή SMTP. 
Εαν χρησιμοποιείτε κάποιο web mail τότε όλα αυτά αντιμετωπίζονται αυτόματα για εσάς και δεν χρειάζεται να ανησυχείτε για τον διακομιστή SMTP που θα χρησιμοποιήσετε. Εαν όμως χρησιμοποιείτε κάποιον mail client τότε πρέπει να εισαγάγετε έναν συγκεκριμένο διακομιστή SMTP ως «διακομιστή εξερχόμενης αλληλογραφίας» (outgoing mail server).

h2. Γιατί να χρησιμοποιήσω τον διακομιστή εξερχόμενης αλληλογραφίας του riseup.net;

Υπάρχουν πολλοί καλοί λόγοι για να χρησιμοποιήσετε τους δικούς μας διακομιστές SMTP:

* Το SMTP που παρέχεται από το riseup.net είναι μέσω κρυπτογραφημένης σύνδεσης. Οι περισσότερες άλλες υπηρεσίες SMTP είναι ανασφαλείς διότι τα μηνύματα αποστέλλονται σε καθαρό κείμενο (clear text).
* Δεν συμπεριλαμβάνουμε τη διεύθυνση IP του σπιτιού σας στις κεφαλίδες της εξερχόμενης αλληλογραφίας σας. Υπάρχουν πολύ λίγοι πάροχοι που το κάνουν αυτό.
* Ορισμένοι πάροχοι υπηρεσιών διαδικτύου δεν σας επιτρέπουν να στέλνετε μηνύματα _from_ διεύθυνση διαφορετική από τα domain τους (για παράδειγμα, εάν προσπαθήσετε να χρησιμοποιήσετε το SMTP της AOL με οποιαδήποτε άλλη διεύθυνση εκτός από τη διεύθυνση aol.com, θα αποτύχει).
* Ορισμένοι πάροχοι υπηρεσιών διαδικτύου (ISP) περιορίζουν σοβαρά τον αριθμό των email που μπορείτε να στείλετε.
* Η υπηρεσία SMTP που παρέχεται από το riseup.net είναι φορητή: δεν εξαρτάται από το δίκτυο στο οποίο είστε συνδεδεμένοι. Θα είστε πάντα σε θέση να το χρησιμοποιείτε ακόμα κι αν μετακινείτε τον υπολογιστή σας.

h2. Πώς μπορώ να ρυθμίσω το SMTP;

Η υπηρεσία εξερχόμενης αλληλογραφίας του Riseup.net απαιτεί έλεγχο ταυτότητας (authentication) και ασφαλή σύνδεση (SSL ή TLS) για λόγους ασφαλείας και anti-spam. Για να χρησιμοποιήσετε το riseup.net ως διακομιστή εξερχόμενης αλληλογραφίας (SMTP), χρησιμοποιήστε αυτές τις ρυθμίσεις:

* *Διακομιστής εξερχόμενης αλληλογραφίας*: @mail.riseup.net@
* *Σύνδεση ή Όνομα χρήστη*: Το όνομα χρήστη που χρησιμοποιείς για να συνδεθείς στο riseup.net.
Για παράδειγμα, εάν η διεύθυνση email σας ήταν joe_hill@riseup.net, η σύνδεσή σας είναι @joe_hill@. Αυτό είναι *απαραίτητο*. Εάν o email client σας δεν υποστηρίζει πιστοποιημένο SMTP τότε δεν μπορείτε να χρησιμοποιήσετε το riseup.net ως το SMTP σας.
* *Χρησιμοποιήστε ασφαλή σύνδεση*: Πάντα.
Αυτό είναι *απαραίτητο*. Εάν ο email client σας δεν υποστηρίζει ασφαλές SMTP τότε δεν μπορείτε να χρησιμοποιήσετε το riseup.net ως SMTP. Μπορεί να έχετε την επιλογή να επιλέξετε *TLS* ή *SSL* για την ασφαλή σύνδεση. Και τα δύο πρωτόκολλα λειτουργούν, αλλά οι περισσότεροι ISP θα μπλοκάρουν τη θύρα 25 (που χρησιμοποιείται από το TLS), γι' αυτό σας συνιστούμε να επιλέξετε SSL.
* *Θύρα*: Για *TLS*, η θύρα πρέπει να είναι @465@. Για *StartTLS*, η θύρα θα πρέπει να είναι @587@. Εάν η μία θύρα είναι μπλοκαρισμένη για εσάς, δοκιμάστε την άλλη.

Για λεπτομέρειες σχετικά με τη ρύθμιση SMTP για διαφορετικούς email clients, ανατρέξτε στην σελίδα [[email clients => email/clients]].

h2. Πόσα μηνύματα μπορώ να στείλω;

Εάν σκοπεύετε να στείλετε χιλιάδες μηνύματα χρησιμοποιώντας SMTP, ίσως χρειαστεί να μας ρωτήσετε πρώτα για να λειτουργήσει. Στείλτε ένα email στο support@riseup.net.

h2. Τι σημαίνει Απαγόρευση Πρόσβασης Αναμετάδοσης (Relay Access Denied);

Το μήνυμα σφάλματος "Relay Access Denied" προκαλείται από ένα από τα ακόλουθα προβλήματα:

# Δεν προσδιορίσατε το mail.riseup.net ως διακομιστή εξερχόμενης αλληλογραφίας.
# Ορίσατε λάθος όνομα χρήστη.
# Δεν έχετε ρυθμίσει τον client σας να χρησιμοποιεί έλεγχο ταυτότητας για την εξερχόμενη αλληλογραφία. Αυτό είναι απαραίτητο.
# Δεν έχετε ρυθμίσει τον client σας να χρησιμοποιεί ασφαλή σύνδεση (TLS ή SSL). Αυτό είναι απαραίτητο.
# Ενεργοποιήσατε τους "ασφαλείς κωδικούς πρόσβασης". Αυτό στην πραγματικότητα δεν είναι πιο ασφαλές, και είναι ασυμβίβαστο με το riseup.net.
# Η βάση δεδομένων των χρηστών μας είναι εκτός σύνδεσης. Αυτό συμβαίνει σε σπάνιες περιπτώσεις, αλλά συνήθως επανέρχεται γρήγορα.

Το πλήρες κείμενο του μηνύματος σφάλματος μπορεί να μοιάζει με αυτό:

<pre>The message could not be sent because one of the recipients was rejected by the server. The rejected e-mail address was 'joe_hill@revolt.org'.</pre>

<pre>Subject 'test', Account: 'myaccount@riseup.net', Server: 'mail.riseup.net', Protocol: SMTP, Server Response: '554 joe_hill@revolt.org: Relay access denied', Port: 25, Secure(SSL): Yes, Server Error: 554, Error Number: 0x800CCC79</pre>

h2. Γιατί μπορώ να στείλω μηνύματα ηλεκτρονικού ταχυδρομείου μόνο σε χρήστες του riseup.net;

Αυτό το πρόβλημα εμφανίζεται όταν δεν έχετε ρυθμίσει τον email client σας να κάνει έλεγχο ταυτότητας στον διακομιστή εξερχόμενης αλληλογραφίας.

h2. Γιατί ο ISP μου μπλοκάρει την εξερχόμενη αλληλογραφία μου;

Υπάρχουν δύο θύρες που μπορείτε να χρησιμοποιήσετε για SMTP:

* Θύρα 465 (TLS)
* Θύρα 587 (StartTLS)

Εάν η μία θύρα είναι μπλοκαρισμένη, δοκιμάστε την άλλη.
